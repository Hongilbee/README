## 1. 명령어 요약 비교

| 명령어 | 구분 | 핵심 기능 | 비고 |
| :---: | :---: | :--- | :--- |
| **top** | 모니터링 | 실시간 시스템 상태 및 프로세스 점유율 확인 | 윈도우 작업관리자와 유사 |
| **ps** | 상태 확인 | 현재 실행 중인 프로세스의 스냅샷 출력 | 특정 시점 확인용 |

## 2. 명령어 상세 조사

### (1) top - 실시간 시스템 모니터링
시스템의 전체적인 운용 상황(CPU, Memory, Swap 등)과 프로세스 현황을 **실시간**으로 갱신하며 보여준다. 서버가 느려지거나 이상 동작을 할 때 가장 먼저 사용하는 명령어이다.

* **주요 출력 정보:**
    * **PID:** 프로세스 ID (Process ID, 고유 식별 번호)
    * **USER:** 프로세스를 실행시킨 사용자 계정
    * **%CPU / %MEM:** CPU 및 메모리 사용량(백분율)
    * **TIME+:** 프로세스가 사용한 토탈 CPU 시간
* **사용법:**
    ```bash
    $ top
    # (종료하려면 'q'를 입력한다)
    ```

### (2) ps (Process Status) - 프로세스 상태 확인
`top`이 실시간 영상이라면, `ps`는 현재 순간을 찍은 **사진(스냅샷)**과 같다. 현재 시스템에서 실행 중인 프로세스의 목록과 상태를 출력한다.

* **주요 옵션:**
    * `-e`: 시스템의 모든 프로세스를 출력한다.
    * `-f`: 풀 포맷(Full-format)으로 상세 정보를 보여준다. (UID, PID, PPID 등)
* **자주 사용하는 조합:**
    ```bash
    $ ps -ef
    # 모든 프로세스를 상세하게 출력함

    $ ps -ef | grep python
    # 'python'이라는 이름이 포함된 프로세스만 필터링하여 검색함
    ```

### (3) jobs - 백그라운드 작업 확인
현재 사용자가 사용 중인 쉘(Shell) 세션 안에서 실행된 **백그라운드(background)** 작업이나 **중지된(stopped)** 작업의 목록을 확인한다.

* **작업 상태(Status):**
    * `Running`: 작업이 계속 실행 중인 상태
    * `Stopped`: 작업이 일시 중단된 상태 (주로 Ctrl+Z 입력 시)
    * `Done`: 작업이 완료된 상태
* **활용:**
    * `+` 기호는 현재 작업(current job), `-` 기호는 이전 작업(previous job)을 의미한다.
    * 확인된 작업 번호를 이용해 `fg`(포그라운드 전환)나 `bg`(백그라운드 실행) 명령을 수행할 수 있다.

### (4) kill - 프로세스 종료 및 시그널 전송
특정 프로세스에 **시그널(Signal)**을 보내 제어한다. 주로 응답이 없거나 불필요한 프로세스를 강제 종료할 때 사용한다.

* **기본 문법:** `kill [옵션] [PID]`
* **주요 시그널 종류:**
    * **-15 (SIGTERM):** *기본값*. 프로세스에게 정상적인 종료를 요청한다. (저장할 시간을 줌)
    * **-9 (SIGKILL):** **강제 종료**. 프로세스의 의사와 상관없이 즉시 종료시킨다. (데이터 손실 위험 있음)
* **주의사항:**
> **[Warning]** `kill` 명령을 사용할 때는 **PID(프로세스 ID)**를 정확하게 입력해야 한다. 실수로 중요한 시스템 데몬을 종료할 경우 시스템이 멈추거나 재부팅될 수 있으므로 주의가 필요함.
